# Analysis Output Directory

## Purpose

This directory is reserved for JSON output generated by the reproducibility script.

Generated analysis output is **not included** in this public repository.

This directory exists only to document the expected output file and how it should be generated if the referenced raw artifacts are later provided to a qualified reviewer.

## Main expected output

* `mdm_false_chain_scan.json`

## How to generate

From the repository root, after raw artifacts have been supplied locally:

`python scripts/analyze_mdm_false_chain.py --root . --out analysis_output/mdm_false_chain_scan.json`

## Expected raw artifact layout

The script expects the following raw ZIP files to exist locally.

These files are **not included** in this public repository.

15G:

* `logs/15G/2026-03-16.zip`
* `logs/15G/2026-03-17.zip`
* `logs/15G/2026-03-18.zip`
* `logs/15G/2026-03-23.zip`
* `logs/15G/2026-03-24.zip`
* `logs/15G/2026-05-05.zip`
* `logs/15G/2026-05-21.zip`

mini1:

* `logs/mini1/mini1-2026-04-03.zip`

## Output contents

The generated JSON may include:

* missing file list
* SHA256 verification
* per-ZIP summary
* important entries
* term counts
* compact hit counts
* observed flags
* case-level summary
* per-date results
* cross-device recurrence indicators

## Important note

The output is not a forensic conclusion.

It is a reproducibility aid.

A reviewer should use the JSON output as an index and then inspect the raw artifacts directly.

## Handling rule

Do not edit generated output manually.

If the script is run again, create a new output file or clearly document that the output was regenerated.

Example:

* `mdm_false_chain_scan_YYYY-MM-DD.json`

## Public repository boundary

Generated JSON output is not included in this public repository.

Raw iOS logs are not included in this public repository.

The output can only be generated after the referenced raw artifacts are provided separately through a secure evidence-transfer process.

## Boundary

The generated output does not establish:

* malware
* payload
* C2
* exploit chain
* APT attribution
* state attribution
* Apple attribution
* criminal attribution
* attacker identity
* MDM enrollment

It supports repeatable review of the selected artifacts only.
